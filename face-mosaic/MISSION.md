# 🎯 APP MISSION — 모자이크 프로젝트 (face-mosaic)

## 한 줄 미션 선언문
사진이 든 폴더를 던지면, 사진 속 모든 사람의 얼굴을 — 특히 신랑·신부 같은 주인공 얼굴을 절대 놓치지 않고 — 자연스럽게 가려서 폴더별로 자동 저장해 주는 내부 일괄 처리 도구.

## 문제 / 동기
현재 사진 속 얼굴을 일일이 수동으로 모자이크 처리하고 있어 시간이 많이 들고 단순 반복 작업이 계속된다. 블로그·SNS에 사진을 올릴 때마다 얼굴 가림 작업이 병목이 된다. 이 반복 작업을 완전 자동화해 처리 시간을 크게 줄이는 것이 목표다.

## 대상 사용자
- 운영자 본인과 직원이 사용하는 **내부 작업 도구** (공개 서비스 아님).
- 사용 환경: **PC 전용** (Chrome · Edge · Arc 권장).
- 모바일/태블릿은 사용 대상이 아니다.

## 핵심 기능 (v1)

### 1. 폴더 입력 (드래그&드롭 또는 폴더 선택)
- 사진이 든 로컬 폴더를 **드래그&드롭**하거나 **폴더 선택** 버튼으로 입력한다.
- **여러 폴더를 동시에** 한 번에 처리할 수 있다 (드래그&드롭 시).
- 대상 포맷: jpg, png 등 일반 사진 포맷.
- 처리 규모 기준: 한 작업에 수십 장(기준치 약 50장), 장당 10MB 내외.
- **기술적 제약**: 브라우저 보안 정책상 임의 경로를 문자열로 직접 입력받을 수 없다. 따라서 "경로 지정"은 폴더 선택 다이얼로그 / 드래그&드롭으로 대체한다. (순수 브라우저 구현 기준.)

### 2. 얼굴 자동 탐지 및 가림 (완전 자동)
- 사진 속 **모든 사람의 얼굴**을 자동으로 탐지한다.
- 식별이 불가능한 수준으로 가리되, **심미성을 유지**한다.
- 하객·제3자 얼굴도 함께 가린다.
- **최우선 원칙: 주인공(신랑·신부) 얼굴 가림을 절대 놓치지 않는다.** 다른 어떤 기준보다 우선한다.

### 3. 가림 스타일 (모자이크 / 블러)
- 가림 스타일은 **모자이크**와 **블러** 두 가지.
- **한 폴더 안의 모든 사진은 같은 스타일로 통일**한다.
- **여러 폴더를 동시에 처리할 때, 각 폴더는 모자이크/블러 중 하나를 랜덤으로 배정**받는다 (폴더 단위 랜덤).
- 심미성 기준: 모자이크/블러에 시선이 과하게 가지 않고 사진 분위기를 크게 해치지 않는, 자연스러운 수준.

### 4. 결과 저장 (원본 보존)
- 처리 결과물은 **각 원본 폴더 안에 `{원본폴더명}_masked` 하위폴더**를 만들어 저장한다. (접미사 `_masked`는 UI에서 변경 가능.)
- **원본 사진은 그대로 보존**한다 (덮어쓰기 없음). 결과물은 JPEG로 저장하며 원본 파일명을 유지한다.

### 핵심 루프 요약
폴더 입력(드래그&드롭 / 경로, 다중 가능) → 모든 얼굴 자동 탐지 → 자연스러운 수준으로 가림(폴더 단위 통일 스타일, 다중 폴더는 폴더별 랜덤 배정) → 각 원본 폴더 안 하위폴더에 결과 저장(원본 보존). **던지면 끝, 결과만 확인.**

## 명시적 비목표 (v1에서 하지 않는 것)
- **검수 / 수동 보정 UI 없음** — 완전 자동 처리만 제공한다.
- **공개 웹서비스 아님** — 내부 도구로만 사용.
- **모바일 미지원** — PC 전용.
- **얼굴 외 객체 자동 가림 없음** — 차량 번호판, 명찰 등은 v1 범위 밖. (추후 확장 후보로만 검토.)

## 측정 가능한 성공 기준
| 지표 | 목표 | 기준 시점 |
|------|------|-----------|
| 주인공(신랑·신부) 얼굴 미가림 | **0건** | v1 / 모든 작업 |
| 약 50장 기준 처리 시간 | **5분 이내 완전 자동 완료** | v1 |
| 결과물 추가 보정 필요 여부 | **추가 보정 없이 바로 블로그/SNS 업로드 가능** | v1 |
| 사용자 개입 | **폴더 입력 후 0회** (던지면 끝) | v1 |

## 명시된 가정
- 출력 하위폴더는 각 원본 폴더 안에 **`{원본폴더명}_masked`** 이름으로 생성되며, **원본은 그대로 보존**된다.
- 입력 포맷은 jpg/png 등 일반 사진 포맷으로 한정한다.
- 처리 규모는 한 작업당 수십 장(약 50장 기준), 장당 10MB 내외를 상정한다.
- "심미성"과 "주인공 얼굴 최우선 가림"은 정성적 기준이며, 실제 결과물로 검증하며 조정한다.
- 주인공 우선 보장은 **"모든 얼굴 누락 0"** 으로 담보한다(전원 가림이 기본이므로 별도 우선 로직 없이, 탐지 민감도를 높여 미검출을 줄이는 방식). 얼굴 미검출 사진은 로그에 경고로 남겨 수동 확인을 유도한다.

## 구현 메모 (v1)
- **스택**: 단일 `index.html` (CDN React + Tailwind) + 얼굴 탐지 `@vladmandic/face-api` (ssdMobilenetv1, WebGL 백엔드) + 로컬 폴더 접근 File System Access API. 보조로 `server.js`(localhost 정적 서버)로 secure context 보장.
- **처리 흐름**: 폴더 핸들 → readwrite 권한 요청 → `{폴더명}_masked` 생성 → 각 사진을 다운스케일(최대 1400px)해 탐지 → 원본 해상도 캔버스에서 얼굴 영역을 타원 클립으로 모자이크/블러 → JPEG로 저장.

## 미해결 / 후속 결정 사항 (Open Questions)
- 모자이크/블러 강도 기본값(현재 약/중/강 선택, 기본 "중") — 실물 결과로 미세 조정 필요.
- 탐지 민감도 기본값(현재 0.35) — 측면·원거리 하객 얼굴 누락 여부 실물 검증.
- (확장 후보) 얼굴 외 객체(차량 번호판·명찰 등) 자동 가림.
