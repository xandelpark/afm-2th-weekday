# 웨딩 커뮤니티 플랫폼 MISSION

> **"플래너가 있는 결혼준비, 거품이없을까요?"**

---

## 1. 미션 선언 (Mission Statement)

우리는 **광고비로 왜곡되지 않은 웨딩 가격 정보를 신부들이 서로 나누는 공개된 족보 플랫폼**을 만든다. 인간 플래너를 소프트웨어와 커뮤니티, 그리고 실제 견적 데이터로 대체하여, 결혼 준비에서 "합리"라는 단어가 적립금 마케팅의 훅이 아닌 실측 가능한 사실이 되도록 한다.

---

## 2. 해결하는 문제 (Problem Statement)

### 2.1 한국 웨딩 시장의 구조적 기만

현재 국내 최대 웨딩 커뮤니티인 **다이렉트결혼준비(네이버 카페)**는 다음과 같은 폐쇄 회로로 작동한다.

1. 업체가 플랫폼에 **광고비·입점비**를 낸다.
2. 플랫폼은 그 돈의 일부를 신부에게 **"적립금"**으로 환원한다.
3. 업체는 광고비를 **견적에 얹어** 신부에게 청구한다.
4. 신부는 **더 비싸게 내고 일부를 돌려받는 착시**를 "합리적 결혼 준비"로 인식한다.

이 구조는 표면적으로 신부에게 혜택을 주는 것처럼 보이지만, 실제로는:

- 가격 투명성을 왜곡하고,
- 적립금이 없는 업체를 고객 시야에서 배제하며,
- 업체 간 실질 경쟁을 광고 예산 경쟁으로 대체한다.

### 2.2 본질적 피해자

신부와 정직한 업체가 동시에 피해를 본다. 신부는 왜곡된 가격 시그널을 받고, 광고비를 감당할 여력이 없는 좋은 소규모 업체는 시장에서 노출되지 못한다.

### 2.3 우리가 겨냥하는 지점

"적립금 = 합리적 결혼 준비"라는 프레임 자체를 해체하고, 그 자리를 **실제 견적 데이터의 공개된 기록(족보)** 으로 대체한다.

## 3. 제품 개요 (Product Overview)

### 3.1 정체성 한 줄 요약

**"플래너 없는 결혼식 커뮤니티"** — 인간 플래너가 수수료로 제공하던 정보 비대칭 해소 기능을, 소프트웨어와 커뮤니티와 데이터가 무상으로 제공한다.

### 3.2 핵심 메커니즘 1: 견적서 족보

- 신부들이 실제로 받은 **웨딩 견적서 이미지**를 업로드한다.
- AI가 이미지를 파싱하여 **사실 정보만 구조화된 데이터로 추출**한다.
  - 업체명, 패키지명, 금액, 날짜, 지역
  - 카테고리별 표준 지표 (대관료·식대·보증인원 등)
- **원본 이미지는 서버에 영구 저장하지 않는다** — 파싱 직후 폐기하거나 24시간 내 자동 삭제한다. 서명·도장·개인정보는 자동 마스킹한다.
- 저장되는 것은 **사실(Fact)만의 집합**이다. 이는 저작권·영업비밀 이슈에 대한 방어선이 된다.
- 참조 모델: 대학 시험 족보 문화, P2P 지식 공유 커뮤니티.

### 3.3 핵심 메커니즘 2: 이중 엔진

| 엔진 | 역할 | 트리거 |
|---|---|---|
| 엔진 1 (Acquisition) | 족보 열람 욕구 → 가입·업로드 유도 | "다른 신부 실견적이 궁금하다" |
| 엔진 2 (Retention) | 포인트 적립 활동 → 일일 재방문 유도 | "포인트 모아서 더 많이 보고 싶다" |

### 3.4 핵심 메커니즘 3: 가상 크레딧 (현금 전환 불가)

다이렉트의 "적립금"과 **정반대 철학**:

- 다이렉트: 돈으로 환원 → 소비주의 강화
- 우리: **정보 접근권으로 환원** → 지식 공동체 형성

참조 모델: Stack Overflow Reputation, Reddit Karma.

**법·세무적 이점**: 전자금융업 등록 불필요, 선불전자지급수단 규제 회피, 세무 부담 최소.

#### 크레딧 획득 표 (초안)

| 활동 | 포인트 |
|---|---|
| 견적서 업로드 | 100p |
| 카테고리 첫 업로드 | 300p |
| 후기 작성 | 30p |
| 질답 채택 | 20p |
| 업체 방문 인증 | 50p |
| 준비 단계 체크리스트 완료 | 10p |
| 주간 연속 접속 | 20p |
| 추천 받음 | 5p |

#### 레벨 시스템

| 레벨 | 조건 | 열람권한 |
|---|---|---|
| 🥉 브론즈 | 견적서 1장 업로드 | 평균 통계 + 개별 견적서 3장 |
| 🥈 실버 | 500p | 업체 실명 + 개별 견적서 10장 |
| 🥇 골드 | 2,000p | 전체 DB 열람 |
| 📗 미슐랭 신부 | 결혼 후 종합 리포트 작성 | 평생 열람권 + 후배 멘토 배지 |

**미슐랭 신부** 제도는 졸업한 신부를 **인플루언서로 전환**하는 장치다. 가장 유경험 사용자가 가장 활발한 바이럴 자산이 된다.

## 4. 차별화 포인트 (Differentiation)

1. **초록 마크(Green Mark)**: 투명 가격 공개에 동의한 동맹 업체에만 부여되는 미슐랭 스타일의 인증 배지. 금전 없이 명예로 작동한다.
2. **AI 파싱 족보**: 다이렉트는 게시글·댓글 기반이라 정량 데이터가 축적되지 않는다. 우리는 **이미지 → 구조화된 수치 DB**. 복제 불가능한 해자.
3. **미슐랭 신부 제도**: 커뮤니티 이탈률이 가장 높은 결혼 직후 시점을 영구 자산으로 전환.
4. **운영자가 업계 내부자**: 업체 관행·꼼수를 콘텐츠로 생산할 수 있다. 외부 기획자는 못 만드는 편집 권위.
5. **광고비 0원 정책**: "우리는 업체에서 돈을 받지 않습니다"가 그 자체로 일관된 브랜드 메시지다.

## 5. 비즈니스 모델 (Business Model)

### 5.1 플랫폼 자체 수익 목표: **0원 (로스리더)**

### 5.2 진짜 수익 구조: 운영자 본업(본식스냅)의 간접 마케팅

- 플랫폼은 본업으로 유입되는 **신뢰 자본**을 생산한다.
- 플랫폼에서 드러나는 운영자의 투명성 철학 자체가 본식스냅 업체 브랜딩의 핵심이 된다.
- "광고비 0원으로 운영되는 커뮤니티를 직접 만든 대표"라는 포지션은 단순 광고 예산으로 살 수 없는 희소 자산이다.

### 5.3 명시적 선언

이 플랫폼은 **수익화하지 않는 것이 전략**이다. 수익화 시도는 플랫폼의 신뢰 자본을 즉시 훼손한다. 이 제약은 버그가 아니라 피처다.

## 6. 참여 업체 모델 (Vendor Participation Model)

업체 카테고리별로 참여 방식을 이중 구조로 설계한다.

### 6.1 공개 동의 동맹 업체 (엄선 모델)

- **본식스냅 5곳** — 운영자 네트워크로 즉시 확보 (운영자 본인 업체 포함)
- **헤어메이크업 5곳** — 본식스냅과 현장 협업하는 팀 단위 확보
- 가격·패키지 투명 공개 동의 → **초록 마크** 부여
- **업체 비용 0원**
- 점진 확장: 드레스샵, 웨딩링, 한복, 청첩장, 신혼여행

### 6.2 웨딩홀: 전체 등재 모델

- **서울·경기 전역 등재** — 제휴비가 없으므로 등재를 제한할 이유도 없다.
- 신부 업로드 견적서에서 **대관료 · 식대 · 보증인원** 3지표만 공개.
- "세부 금액은 계약 시기에 따라 차이 있음" 면책 문구 고정.
- 동맹 여부와 무관하게 DB를 구축하여 **트래픽 입구**로 활용.

### 6.3 향후 미동맹 업체 설득 루트

초기 단계에 축적된 조회수 데이터("귀사 페이지 월 조회 300건")를 미동맹 업체 설득 자료로 전환. 업체는 돈을 내지 않고, 다만 가격 공개에 동의한다.

## 7. 데이터 표준 (Data Standards)

각 카테고리마다 3~5개의 핵심 지표만 표준화 공개한다. 나머지 세부 변동은 "계약 시기에 따라 차이 있음" 면책으로 일괄 처리한다.

| 카테고리 | 표준 지표 |
|---|---|
| 예식장 | 대관료 · 식대 · 보증인원 |
| 스튜디오 | 촬영 횟수 · 앨범 사양 · 원본 제공 여부 |
| 드레스 | 본식 드레스 벌수 · 피팅 횟수 · 대여 기간 |
| 메이크업 | 본식 · 리허설 · 혼주 포함 여부 |
| 본식스냅 | 촬영 인원 · 원본 매수 · 편집 컷 수 |

## 8. 이해상충 관리 (Conflict-of-Interest Governance)

운영자가 본식스냅 업체 대표라는 사실은 **약점이 아닌 강점으로 설계**한다. 단, 다음 원칙을 약관에 명시한다.

- 본식스냅 동맹 5곳 중 **운영자 본인 업체 포함** (과잉 투명 원칙).
- 플랫폼 내 운영자 본인 업체에 대한 **상단 노출·추천 알고리즘 우대 절대 금지**.
- 운영자 본인 업체에 대한 후기·평가도 다른 업체와 동일한 필터 없이 표출.
- "투명성 철학상 운영자 본인이 가장 먼저, 가장 투명해야 한다"는 원칙을 약관 서문에 명기.

## 9. Go-to-Market 전략 (Phased Launch)

### Phase 0 — 오픈 전 (2~3개월)

- 운영자가 본식스냅 5곳, 헤메 5곳 동맹 확보.
- 서울·경기 웨딩홀 DB 수동 큐레이션 (공개 정보 기반).
- AI 파싱 파이프라인 구축 (GPT-4V / Claude 등 외부 모델 활용).
- 서비스명·도메인 확정.
- 약관, 개인정보처리방침, 이해상충 고지문 정비.

### Phase 1 — 런칭

- 신부 가입 → 웨딩홀 견적서 업로드 → 크레딧 획득 → 동맹 업체 정보 열람의 기본 루프 가동.
- 초록 마크 시스템 공식 런칭.
- 커뮤니티 게시판·Q&A·체크리스트 기능 개방.

### Phase 2 — 확장

- 드레스·웨딩링 카테고리 동맹 확장 (번들 세일즈 전략).
- 미동맹 업체에 대한 익명 견적 수집 (지역+등급만 표시).
- "귀사 페이지 월 조회 N건" 데이터를 활용한 미동맹 업체 설득.
- 미슐랭 신부 1기 인증 시작.

## 11. 경쟁 우위 / 해자 (Competitive Moat)

1. **업체 광고비 0원** → 가격 왜곡 없음. 다이렉트가 이 구조로 전환하면 기존 수익 모델이 즉시 붕괴하므로 모방 불가.
2. **운영자가 업체 대표** → 업계 내부 관행 폭로·해설 콘텐츠 생산 가능. 외부 기획자 영입으로 복제 불가.
3. **AI 파싱 족보** → 정량적 가격 증거가 시간에 따라 복리로 축적. 후발 주자는 데이터 공백을 메울 방법이 없음.
4. **초록 마크 생태계** → 업체가 돈이 아니라 명예를 유인으로 참여. 복제 시 신뢰가 무너지는 역설적 장치.
5. **미슐랭 신부 제도** → 졸업 유저를 바이럴 자산으로 전환. 일반 커뮤니티의 구조적 이탈을 무력화.

## 12. 위험 요소와 대응 (Risks & Mitigations)

| 위험 | 대응 |
|---|---|
| 업체의 법적 항의 (견적 공개, 명예훼손) | 원본 이미지 미저장 + 사실(Fact)만 추출 + 면책 문구 + 분쟁 시 해당 데이터 즉시 삭제 프로토콜 |
| 개인정보 유출 우려 | 업로드 시 서명·도장·개인정보 자동 마스킹, 원본 24시간 내 폐기, 법률 검토 필수 |
| 업체 반발·보이콧 | 동맹 업체 5+5 선확보로 최소 런칭 기반 확보, 이후 조회수 데이터로 설득 |
| 크레딧 어뷰징(허위 견적 업로드) | 업로드 검증 큐 + 커뮤니티 신고 + 중복 탐지 + AI 이상 탐지 |
| 운영자 본업 업체에 대한 역풍 | 과잉 투명 원칙 공개, 약관에 알고리즘 우대 금지 명기, 필요시 외부 감사 |
| 다이렉트의 모방·소송 | 다이렉트가 광고비 모델을 포기할 수 없다는 구조적 비대칭이 방어. 상표·명예훼손 소송 대비 법률 자문 사전 확보 |
| AI 파싱 오류에 의한 잘못된 정보 | 파싱 결과 사용자 확인 단계 필수, 수정 요청 기능, 오탐률 지속 모니터링 |

## 부록 A. 서비스명 후보 (제안)

현재 서비스명은 미정이며, 슬로건 ("플래너가 있는 결혼준비, 거품이없을까요?") 철학과 일관되는 후보를 제안한다.

1. **거품없는결혼** — 슬로건 직결, 메시지 강도 최대, SEO 우위
2. **웨딩족보** — 메커니즘 직설, 기억 용이, 업계에 없던 포지셔닝
3. **맨얼굴웨딩** — 투명성의 감성적 은유, 브랜딩 여지 큼
4. **원가웨딩** — 강력하지만 법적 리스크 가능성 있음(원가 주장의 입증 부담)
5. **플래너없이** — 플랫폼 정체성 직설, 도메인 확보 용이성 검토 필요

최종 결정은 도메인 가용성·상표 검색·운영자 감각을 종합하여 별도 의사결정.
