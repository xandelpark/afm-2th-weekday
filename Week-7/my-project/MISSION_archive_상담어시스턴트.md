# 본식스냅 상담 어시스턴트 (Wedding Snap Consult Assistant)

## 한 줄 요약
본식스냅 업체 사장님이 직접 답변해 온 "정답 없는 가치판단형 고객 질문"의 과거 답변을 상담사 직원이 실시간 검색해 사장님 판단의 결을 그대로 반영해 응대할 수 있게 돕는 사내 검색 도구.

## 누구를 위한 것인가 (페르소나)
- 본식스냅 업체에 고용된 전담 상담사 직원
- 카카오채널로 들어오는 고객 문의에 응대하는 게 업무
- 기본 안내(가격표, 일정, 표준 옵션)는 잘 처리하지만 비정형·재량 판단이 필요한 질문에서 막힘
- 막힐 때마다 사장님께 묻는 게 부담

## 무엇을 하는가
- 상담사가 검색창에 고객 질문(또는 키워드)을 입력
- 시스템이 과거 유사 Q&A 3-5건을 즉시 표시
- 상담사는 과거 답변에서 "사장님의 판단 결"을 읽고 자기 답변에 반영

## 왜 (해결하는 문제)
- 가치판단형 질문에서 사장님이 매번 직접 답해야 하는 시간 손실
- 상담사가 즉답 못 해 응답 지연 → 고객 이탈 위험
- 사장님 부재 시 상담 품질이 떨어지는 의존 구조

## 핵심 가치 (다른 챗봇과 무엇이 다른가)
- "정답"을 주지 않는다 — 사장님의 판단 결을 **참고 자료**로 보여줄 뿐
- LLM이 답을 지어내지 않는다 — 잘못된 톤/판단으로 신뢰가 깨지지 않게
- 최종 결정은 항상 사람(상담사)이 한다

## MVP 성공 정의 (측정 가능)
- 사장님이 본인 카톡 export에서 정제한 시드 Q-A **30건 이상** 입력 완료
- 상담사 테스트에서 **새 질문 5개 중 3개 이상**에 적절한 과거 Q&A가 검색됨
- 검색 결과가 **2초 이내** 표시
- 상담사가 별도 웹페이지에 접속해 검색하고 결과를 보는 흐름이 처음부터 끝까지 막힘 없이 동작

## 명시적 비범위 (Out of Scope — v2 이후)
- LLM이 답변 자동 생성 (사장 톤 흉내)
- 카카오채널/상담톡 API 자동 연동
- 사장님 피드백 루프 (새 답변 추가/수정 UI)
- 다국어 지원
- 모바일 앱 / 카톡 내장
- 사용자 권한/계정 분리 (MVP는 단일 사내 도구)

## 데이터 부트스트랩 절차 (1회성)
1. 사장님이 PC카톡에서 주요 1:1 채팅방 5-10개를 .txt로 export
2. 사장님이 그 중 "상담사가 막혀 본인이 답한 가치판단형 Q&A"만 30-50건 추려냄
3. 관리자 화면(또는 SQL 직접 입력)으로 DB에 시드

## 기술 스택 메모
- Backend: Node.js + Express, 단일 `server.js`
- DB: Supabase PostgreSQL (`pg` 드라이버)
- Frontend: React + Tailwind CSS (CDN, 단일 `index.html`)
- 검색: 1차로 PostgreSQL Full-Text Search 한국어 (또는 ILIKE) — 부족하면 임베딩 + pgvector로 업그레이드 (dev-kickstart 단계에서 결정)
- 배포: 로컬 → 필요 시 Vercel
